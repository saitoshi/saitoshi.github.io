<p> Sudoku is a logic based, combinatorial, number placing game. In the UH Manoa's ICS 211 course, as a part of the recursive lesson plan, students including me created a recursive Hexadecimal sudoku solver based off of the given directions from Professor Cam Moore. The directions for the assignment can be found <a href = "http://courses.ics.hawaii.edu/ics211s18-4/morea/110.recursion/experience-H08.html"> here </a>. </p>

<p> The assignment was done alone as majority of the work in ICS 211 is not group work. We were given a week to do this and the main goal was to create a program that checks a given sixteen by sixteen sudoku grid and checks for possible solutions. More the sudoku is filled meant that the possible solutions could be computed quickly but a sudoku grid that was close to being empty took a long time. A junit test was created to test the sudoku grid and only once the junit test passed is when the assignments were allowed to be turned in. The codes used for the solver can be found using this <a href = "https://github.com/saitoshi/sudokurecursivesolver"> link </a> </p>

<p> From this assignment, I learned recursive and more importantly how the right computer environment is really needed. Half of the reason that the solver took a long time was because of the computer I had could not handle the work. Therefore, I had to borrow someone else computer to test it out. </p>
